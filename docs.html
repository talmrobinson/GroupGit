Henryola Documentation

Henryola Documentation Henryola is an a free web application that assists those with an UCSC email address in scheduling private study rooms at the Mchenry Library. Henryola makes the tedious process of obtaining room appoints easy and lets you spend more time with your studies. The Mchenry has their own site for booking rooms but it requires that you check the availability of each room individually per day. Furthermore, come time book, one may only book one room at a time. Henryola gives you a broad view of both the entire week and all of the rooms on every floor. Getting Started

Installation and User Guide

Getting started with Henryola is easy. So easy in fact that it was designed with hopes of being intuitive enough to not need many onscreen instruction or user’s manuals.

Simply navigate to our URL.

From our home screen you will be prompted to enter your information such as name and @ucsc.edu address. Do not worry, we are not collecting any of that data, we are simply giving it to the library in order for their room booking tool to work.

Afterwards you will be shown a grid representing a calendar for the week. Drag, tap, click the cells to select times that you would like to have the room booked. Cells with the color blue indicate available time slots. When you have filled your availability, click next.

The next screen is the same grid but this time it is showing you what time slots are available by showing the time in purple. Browse and select the rooms and times you’d like to have booked. The rooms staged to be booked for you will turn dark blue. If you see only gray squares on all rooms, then there were no available times that worked with your preferences. When you have your desired rooms selected click book.

And Henryola! You successfully booked a room. To confirm your booking, access your UCSC email inbox and confirm/deny your appointment within 2 hours. Note: There is a delay (approx 20 - 40 seconds) for receiving the email from the school server. Simply refresh your inbox until you see the email.

Overview

Henryola can essentially be broken into 3 large pieces which will be discussed in greater detail

The server/hosting of the site: pertains to all technologies implemented to make allow our site to exist on the web, our url handlers, and our post handlers. We used www.glitch.com to host our website. Glitch is a free website that lets you deploy web applications using NodeJs. You are given a URL and a text editor that your whole team can edit in. Projects can be previewed and then pushed to git. Glitch had a tutorial book called “The Node Beginner Book” by Manuel Kiessling https://node-beginner.glitch.me. We used this book as a framework for setting up our index.js and router. Node.js As our Javascript framework for handling server side programming. The Backend files are as follows: index.js, myAJAXCalls.js, package.json, requestHandlers.js ,router.js, and server.js

The User Interface: pertains to how the user interacts with our site. The UI encompases the user experiance from opening the index of the website to the final booking result. Code under the UI umbrella involves how the user enters information in a way that is easy for them and useful to our program, the visual appeal of the website and the functionality of navigating the site. These files include: Docs.html, index.html, multistep.js, style.css

The Screen Scraping and headless browsing: Pertains to the libraries and algorithms used to both get availability of rooms from the library and book the rooms with the user's preferences. The availability uses a javascript library called cheerio for screen scraping the right tag to find available rooms. The book functionality is handled by a NodeJS library called node-horseman which allows for a headless browser post requests. The users preferences are stored in a boolean array. The Mchenry’s available room times are stored in an identical array. The two arrays are then ANDed to see where there exists overlap. These files include: availTool.js, bookTool.js

index.html

In the case of Henryola, the entire application is run in the browser from the single index URL, therefore our index is responsible for all the html in which the user will enter information and it calls the files/functions to stylize the page such as multistep.js,style.css, and the calendar canvas from UI.js. multistep.js

multistep.js contains the javascript code that adds the multistep functionality to the Henryola UI. (The code is adapted from https://codepen.io/designify-me/pen/qrJWpG.)

style.css

Style.css simply styles the HTML page.

UI.js

UI.js performs three main tasks that are fundamental to receiving and handling user input. These tasks include getting user time availability, getting open Mchenry's room availability, and finding the intersection of those two sets to allow the user to choose what times they would like to book. It also defines the behavior for our random time slot feature, that books a random time slot from the available spaces.

availTools.js

availTools.js does most of the heavy lifting in terms of scraping the mchenry website for availability. The bulk of which happens in the function getAvail(). This function essentially takes the availability for each prospective room in mchenry, and represents it in a 7 by 32 array. This is then packaged into a json object through a series of helper function calls, that ends with jsonify().

myAJAXCalls.js

myAJAXCalls.js holds the two ajax calls that deal with getting room availability, and booking a room. As soon as the webpage is loaded, myAJAXCalls requests room data, which is then used by UI.js later.
